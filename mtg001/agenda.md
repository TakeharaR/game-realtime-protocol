# 1. 当会の方針に関する議論

### 活動目的

ゲーム業界においてリアルタイム通信で使用されるプロトコルについて情報交換を行い、業界全体でリアルタイム通信の品質向上を目指す。


### 活動

1. インターネット上での活動(利用サービスについては後述)
2. リアルでの会合(開催頻度は後述)
3. CEDEC 等のカンファレンスでの報告


### 活動場所について

将来的に標準を目指すのであればオープンな場での議論が必要だが、現状 Discord は活発と言えず、オープンな場では話し辛い空気がある。
招待制の Slack 等でまずは議論を醸成してから、こなれてきたらオープンな活動に移行する方針としたい。

以下上記 Slack も含めた案。

1. Slack (招待制) に移行する
2. X 及び GitHub での議論で十分
3. このまま Discord でいく
4. その他


### 会合について

- 頻度
    - 年三回
- タイミング
    - IETF の一か月後
- 開催形式
    - リアルとオンライン(Zoom)のハイブリッド
- 開催日
    - 平日 (仕事として参加し易いのを重視)
- 議題
    - IETF であった当会の関連事項の報告
    - 事前に挙がった議題に関する協議
    - その他関連した発表や情報共有(あれば)
- その他
    - 会合の後に交流会を実施


### Milestone

1. ゲーム業界のリアルタイム通信プロトコルにおける課題を洗い出す (イマココ)
2. QUIC 等標準のプロトコルを用いた場合に上記課題が解決できるか、解決できないものは何かを洗い出す
    - タイミング的に採用事例も出始める頃だと思うので、その辺りも参考に進める
3. 2 の問題点を解決する為に必要なものは何かを議論する
    - QUIC の拡張として定義すれば事足りるのか
    - 根本的に別のプロトコルが必要なのか
4. (必要であれば) 3 の要素を取り入れたゲームのリアルタイム通信プロトコルの標準化を進める
